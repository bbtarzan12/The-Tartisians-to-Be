# 01. 게임 개요

## 장르

**3D 뱀파이어 서바이벌류** — 대량의 적이 동시에 등장하는 액션 로그라이트.

플레이어는 사방에서 몰려드는 적의 무리 속에서 **자동 전투**로 버티며, 적을 처치해 얻은 경험치로 **레벨업 → 무기·능력 강화**를 반복하고, 제한 시간(목표 5분) 생존을 노린다. *Vampire Survivors*, *20 Minutes Till Dawn* 계열을 3D로 재해석.

## 핵심 게임 루프

```
이동(WASD) → 적이 플레이어를 추적 → 무기 자동 발사 → 적 처치
   → 경험치 젬 드랍 → 흡수 → 레벨업 → 업그레이드 3택 선택
   → 더 강해짐 → 더 많은 적 → ... → 5분 생존 시 클리어 / 사망 시 게임오버
```

## 디자인 필러 (이 장르가 옳아야 하는 3가지)

프로토타입이라도 다음 3개는 처음부터 옳게 설계한다:

1. **대량 엔티티 성능** — 동시 적 100~200체 + 다수 VFX가 프레임 드랍 없이 굴러간다. 여기서 무너지면 장르가 성립하지 않는다.
2. **데이터 주도 설계** — 무기·적·업그레이드가 수십~수백 종으로 늘어난다. 코드가 아니라 데이터(ScriptableObject)로 정의해야 확장·밸런싱이 가능하다.
3. **시뮬레이션 / 렌더링 분리** — 게임플레이 로직과 렌더·물리를 분리해 나중에 리플레이·자동 밸런싱·성능 교체가 열려 있게 한다.

## 규모와 타깃

| 항목 | 결정 |
|------|------|
| 플랫폼 | **PC** (모바일·콘솔 비대상) |
| 동시 적 수 | **최대 100~200체** |
| VFX | 다수 (사망 폭발·피격·오라 등) |
| 렌더 파이프라인 | URP (Deferred+) |

> 이 규모 결정이 기술 선택을 좌우한다 — 200체 규모에서는 DOTS/ECS/Jobs가 과설계이며, 평범한 GameObject + 풀링이 정답이다. 자세한 근거는 [03. 기술 스택](03-tech-stack.md) 참조.

## 프로토타입의 목표

핵심 재미(자동 전투 × 성장 루프 × 생존 긴장감)를 **전문가급 기반 위에서** 검증한다. "프로토타입이니 대충"이 아니라, 그대로 본 개발로 이어갈 수 있는 확장 가능한 골격을 만든다.

## 비범위 (이번 프로토타입에서 하지 않는 것)

- 멀티플레이, 세이브/로드, 메타 progression(영구 강화), 상점
- 정교한 아트 에셋 (당분간 프리미티브 플레이스홀더)
- 모바일 최적화, 콘솔 이식
