# 07. GOAL (자율 실행 목표 정의)

`goal` 기능으로 자율 루프를 구동하기 위한 **목표 명세**. 헌장(`docs/06`, "어떻게")과 로드맵(`docs/05`, "무엇을")을 묶어, 루프가 **매 반복마다 무엇을 향하고, 언제 끝났다고 판정하는지**를 결정론적으로 정의한다.

---

## 한 줄 목표 (Goal Statement)

> `docs/05` 로드맵 M1~M6를 자율 실행 헌장(`docs/06`) 규칙 아래 전부 완료하여, **시작 → 플레이 → 레벨업 → 5분 생존이 끊김 없이 완결되는, 플레이 가능한 전문가급 3D 뱀서류 프로토타입**을 만든다.

## 성공 기준 (DONE — 루프 종료 조건)

아래를 **전부** 충족하면 GOAL 달성 → 루프 종료:

1. M1~M6 **모든 마일스톤이 완료 정의(DoD) 충족** 및 각각 커밋됨
2. **통합 플레이스루**: 한 판이 시작부터 5분 생존(승리) 또는 사망(패배)까지 끊김·예외 없이 완결 — MCP 플레이모드 + 스크린샷으로 증빙
3. **성능 게이트**: 동시 적 200체에서 60fps 유지 — 프로파일러로 증빙
4. **전체 테스트 스위트 그린** + 컴파일 에러/런타임 예외 0
5. docs가 구현과 동기화됨

## 진행 단위 & 상태 추적 (컨텍스트가 끊겨도 재개 가능하게)

- **진행 단위 = 마일스톤 6개.** 각 마일스톤은 task 묶음으로 표현.
- **"지금 어디인가"는 결정론적으로 판정한다:**
  - `git log` 의 **마일스톤별 커밋**(예: `M1: 기반 골격 …`)
  - task 리스트 상태
  - `docs/05` "현재 위치" 섹션
- 세 신호가 일치하도록 매 마일스톤 완료 시 함께 갱신한다. 이로써 컨텍스트 압축·세션 재시작 후에도 "마지막 완료 마일스톤 다음"을 명확히 집어낼 수 있다.

## 매 반복(iteration) 동작

```
1. 상태 판정: git 로그·task·docs로 마지막 완료 마일스톤 확인
            → 다음 마일스톤 N 선택 (없으면 GOAL 완료 → 종료 보고)
2. 구현: docs/05 의 N 스펙대로 스크립트·SO·데이터·씬·프리팹 작성
3. 검증 루프: 작성 → assets-refresh → 콘솔 에러 0 확인
            → tests-run 통과 → 플레이모드/성능 실측 검증
4. DoD 판정:
   - 충족 → docs/task 갱신 → 자동 커밋(한국어, "MN: …") → 완료 보고 → 1로
   - 미충족 → 자가수정(최대 3회) → 그래도 실패면 STOP & 보고
5. STOP 조건(헌장 §5) 발생 시: 멈추고 사용자에게 단일 명확 질문
```

## 불변 제약 (반복 내내 항상 참)

- 코딩 컨벤션(`docs/04`): Unity 6.2 **C# 9.0**, 모던 패턴, 핫패스 할당 제로
- 모든 스폰 객체(적·투사체·젬·VFX) 풀링
- **빌드는 항상 그린** — 깨진 상태로 다음 단계 진입 금지
- 커밋은 **검증 통과 후에만**
- 잠긴 결정(`docs/06` §2) 변경 금지 — 바꾸려면 STOP & 질문

## STOP 조건 (요약, 상세는 헌장 §5)

게임 디자인 분기 · 비가역/파괴적 작업 · 동일 오류 3회 자가수정 실패 · 합의 외 의존성 · 범위 이탈.

## 종료 시 산출물 (Deliverables)

- 플레이 가능한 프로토타입 (Bootstrap→Game 씬 한 판 완결)
- 마일스톤별 커밋 히스토리 (한국어)
- 구현과 동기화된 `docs/`
- 통과하는 EditMode/PlayMode 테스트 스위트
- **최종 요약 보고**: 만든 것 · 성능 수치 · 알려진 한계 · 다음 권장 작업

## 루프 운영 (페이싱)

- 자기 페이싱: 한 마일스톤 완료 후 즉시 다음으로. 사용자 인터럽트 항상 가능.
- 컴파일/임포트 대기 등은 짧은 간격으로 폴링, 긴 유휴는 길게.
- 비결정적 시각(시계/난수)에 의존하지 않고, 위 상태 신호로만 재개 위치를 판정.

## 비-목표 (이번 GOAL 범위 밖)

멀티플레이 · 세이브/로드 · 메타 progression · 정교한 아트 · 모바일/콘솔 이식 · 사운드 디자인 (플레이스홀더 이상).
