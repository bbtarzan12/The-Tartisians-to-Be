# 08. 현재 빌드 기획서 (As-Built)

> 지금 **실제로 구현·동작하는** 게임을 역으로 정리한 문서다. 수치는 코드/에셋의 현재 값이며, 밸런싱·확장 논의의 기준점으로 쓴다. (앞을 내다보는 계획은 `05-roadmap.md`, 이 문서는 "지금 상태")
> 기준 커밋: `fad6cee` · 테스트 70/70 · 200체 @193fps

---

## 1. 한 줄 요약

탑다운 쿼터뷰 3D 뱀파이어 서바이벌류. 플레이어는 WASD로 움직이며 **무기는 자동 발사**, 몰려드는 적을 처치해 **경험치 젬**을 모으고, 레벨업마다 **업그레이드 3택**으로 강해지며 **5분 생존**을 노린다.

## 2. 조작

| 입력 | 동작 |
|------|------|
| WASD / 방향키 / 게임패드 좌스틱 | 플레이어 이동 (XZ 평면) |
| (자동) | 무기 발사 — 사거리 내 최근접 적을 자동 조준 |
| 마우스 클릭 | 레벨업 카드 선택 / 종료 화면 "다시 시작" |

## 3. 게임 플로우

```
시작(Game 씬) → 플레이(자동전투·생존)
   ├─ 적 처치 → XP 젬 드랍 → 자석 흡수 → 경험치
   │     └─ 레벨업 → [일시정지] 업그레이드 3택 → 선택 → 재개 (강해짐)
   ├─ 플레이어 HP 0 → 게임 오버
   └─ 5분 생존 → 생존 성공(승리)
종료 화면 → "다시 시작"(씬 리로드)
```

상태 기계(`GameDirector`): `Playing / LevelUp / GameOver / Victory`. LevelUp·종료 시 `Time.timeScale = 0`.

## 4. 플레이어

| 스탯 | 값 | 비고 |
|------|----|----|
| 최대 체력 | 100 | `PlayerDefinition` |
| 이동 속도 | 6 | 런타임 `RunStats`로 업그레이드 반영 |
| 자석 범위(pickupRadius) | 2.5 | 젬 흡수 반경 |
| 이동 방식 | Rigidbody(키네마틱) MovePosition | XZ 평면, 대각 정규화 |
| 이동 경계 | 아레나 ±59 clamp + 장애물 SDF 밀어내기 | 둘레 벽 + 내부 장애물 비침투(적과 동일 `ObstacleField`) |
| 비주얼 | **Crystal 스킨드 캐릭터** + Idle/Walk 애니메이션 | 아래 캐릭터/애니메이션 참고 |

- **캐릭터/애니메이션(Mecanim):** `Player` 하위 비주얼에 **Crystal 스킨드 메시**(Generic 리그 5본) + Animator. `Crystal.controller`의 **1D 블렌드 트리(`Speed`)** 가 Idle↔Walk를 섞고, `CharacterVisual`이 부모 이동량으로 `Speed`(0~1)를 구동 + 진행방향으로 회전(이동 로직과 분리된 표현 셸 → 입력 주체가 누구든 동일 동작). 루트모션 off(이동은 코드 구동).
  - **Generic 멀티-FBX 정석:** 메시 FBX가 아바타(`CrystalAvatar`) 생성 → 애니 FBX(Idle/Walk)는 **Copy From Other Avatar → CrystalAvatar**로 아바타를 공유해야 클립이 메시에 정확히 바인딩(미통일 시 T포즈). 텍스처 8K→2K(Base sRGB·Normal), 머티리얼 `Crystal.mat`(URP/Lit).

## 5. 카메라

Cinemachine 쿼터뷰. 플레이어 추종(Follow) + 항상 바라봄(HardLookAt). 오프셋 `(0, 14, -12)` → 약 45° 내려다보기.

## 6. 적 (3종)

스폰 시 플레이어 주변 반경 18 링에서 등장 → 중앙 시뮬레이션이 흐름장(seek) + **PBD 군중 솔버**로 이동. 접촉 시 플레이어에 초당 데미지.

| 종류 | 체력 | 이동속도 | 접촉 DPS | XP | 크기(scale) | 성격 |
|------|------|---------|---------|----|-----------|----|
| Chaser | 10 | 3.0 | 5 | 1 | 1.0 | 기본 |
| Swift | 6 | 5.0 | 3 | 1 | 0.7 | 빠르고 약함 |
| Brute | 30 | 1.8 | 10 | 3 | 1.5 | 느리고 단단 |

- 비주얼: 빨간 캡슐(공유 머티리얼 → GPU 인스턴싱 배칭)
- AI: `EnemySimulation`이 매 FixedUpdate에 전체 적을 한 번에 갱신(데이터 지향, 얇은 셸). 이웃 질의는 `SpatialHashGrid`(물리 미사용).
- **군중 이동(PBD 통합 솔버):** `CrowdSolver` — Position-Based Multi-Agent Dynamics(Weiss/Karamouzas, MiG 2017) 정석 구현. 과거의 휴리스틱 더미(힘 separation·벽 접선 투영·개별 위치 보정·하드 클램프·step clamp)를 **하나의 위치 제약 투영**으로 통일. 흐름:
  1. 속도 블렌딩 `v=lerp(v_prev, v_pref, α)` → 예측 위치 `x*=x+v·dt`
  2. 이웃 탐색(예측 위치 기준) → 안정화 1회 + 주반복 6회 Jacobi delta-averaging(SOR 1.2)
  3. 제약: **적-적 비침투(마찰 접촉) + 적-벽 비침투(무한 질량, 동일 제약) + 장거리 예측 회피(time-to-collision)**
  4. 속도 복원 `(x*-x)/dt` → XSPH 점성 → 최대 가속/속도 클램프
  - **'밀지 않음 / 공간 없으면 정지 / 벽 비침투'가 하드 제약에서 자연 발생**(별도 if-휴리스틱 없음). 적-벽과 적-적이 같은 제약 형태.
  - 검증(100마리×200스텝): 벽 침투 0 · 적층 없음(최근접쌍 0.855) · 속도 튐 없음(2.19/cap 4) · 벽에 붙은 목표에 끝까지 접근(최근접 0.434).
- **벽 충돌(해석적 SDF):** `ObstacleField` — 축정렬 박스(AABB) 집합의 해석적 거리·법선. 격자 거리장과 달리 **연속**이라 벽 근처 떨림이 없다(격자 SDF 대비 변위 표준편차 0.150→0.000). `FlowFieldController`가 `NavObstacle` 박스로 구성해 `ServiceLocator`에 등록 → 솔버의 벽 제약에 공급.
- **네비게이션(NavMesh 미사용):** `FlowField`(그리드 비용장→방향장, BFS) — **선호속도(전역 라우팅) 소스**로만 사용(충돌 아님). `FlowFieldController`가 영역(120×120, 셀1)·장애물(`NavObstacle`)을 차단 셀로 표시하고 플레이어 셀 변경 시 재계산. **윈도우 BFS**: 목표 주변 반경 40셀 창만 계산 → 적은 항상 플레이어 근처(스폰 ≤28)라 창 밖 불필요, **재계산 비용이 맵 크기와 무관하게 일정**(120²에서 윈도우 1.98ms vs 전체 4.2ms, 에디터). 창 밖은 직선 추적 폴백. 디버그 기즈모 토글 제공.
- **아레나:** 둘레 벽 4개(NavObstacle, ±60, 내벽면 ±59, 두께 2)로 120×120 폐쇄 공간 → flow 격자(120×120)와 일치. 바닥 120×120. 플레이어는 ±59 clamp + 장애물 SDF로 항상 격자 안.
- **레벨 디자인(ProBuilder, 미로 아님):** 열린 중앙 + 흩어진 석재 구조물 12개(`Obstacles` 하위, 공용 머티리얼 `ObstacleStone`) — 코너 모놀리스 4(높이5, C2 회전대칭) + 변 슬래브 4(높이3) + 내부 기둥 4(4×4, 높이6). 전부 축정렬 박스(AABB 충돌·흐름 차단 정확). 넓은 통로·카이팅 피벗, 흐름 라우팅 showcase. 모두 `NavObstacle`이라 적·플레이어가 우회·충돌.
- 접촉 데미지: 적이 접촉 반경 내면 플레이어에 초당 데미지 적용.

## 7. 스폰 (Wave_Default)

| 항목 | 값 |
|------|----|
| 스폰 간격 | 0.15초 |
| 동시 최대(maxAlive) | 200 |
| 스폰 거리 | 18~28 링(SpawnRadius~+depth), 아레나로 clamp |
| 등장 종류 | Chaser / Swift / Brute 무작위 |

- **스폰 위치(화면 밖 + 아레나 안):** 플레이어 주변 링에서 각도·거리를 뽑아 **아레나(±18)로 clamp** → **카메라 화면 밖**(뷰포트 밖, 여유 4%)이고 **장애물 밖**이며 플레이어로부터 ≥9인 위치만 채택(거부 샘플링 24회). 검증: 200샘플 화면 안 0·아레나 이탈 0·장애물 0.
> 현재는 단일 웨이브가 일정 간격으로 maxAlive까지 무작위 스폰. 시간대별 난도 상승·보스는 미구현.

## 8. 무기 (MagicBolt · 1종)

| 항목 | 값 |
|------|----|
| 발사 간격 | 0.6초 |
| 데미지 | 5 |
| 투사체 속도 | 14 |
| 관통 | 0 (첫 명중 후 소멸) |
| 사거리 | 12 |
| 투사체 수명 | 2초 |

- 동작: 사거리 내 **최근접 적** 자동 조준, **적 중심 높이에서 발사**(키 작은 적도 명중). **예측 사격**(적 속도 기반 선행 조준, leadFactor 0.6). 투사체는 트리거 충돌로 데미지.
- **시야 차단(LOS):** **벽·장애물 너머의 적은 조준하지 않는다** — `ObstacleField.Blocks`(선분-AABB 교차)로 차단된 적은 건너뛰고 다음 후보를 조준.
- **투사체 벽 충돌:** 투사체가 벽·장애물에 닿으면(해석적 `ObstacleField` 거리 < 0.2) 소멸(풀 반환). 적 충돌과 별개.
- 비주얼: 노란 구체, 풀링.

## 9. 전투·사망

- 데미지 단일 진입점 `DamageSystem`. 적/플레이어 공용 `Health`(`IDamageable`).
- 적 사망 → **사망 VFX**(VFX Graph `DeathPoof`, 풀링) + `EnemyDiedEvent` 발행 → 젬 스폰.

## 10. 진행 (경험치·레벨·업그레이드)

- **경험치 곡선:** 다음 레벨 요구량 = `5 + (레벨-1)×3` (L1→5, L2→8, L3→11 …). 초과분 이월.
- **XP 젬:** 사망 위치 드랍 → 플레이어 pickupRadius 내면 자석으로 끌려와 흡수.
- **레벨업:** 일시정지 + 업그레이드 **무작위 3택**(중복 없음) → 선택 적용 → 재개. 다중 레벨업은 연속 카드.

## 13. 기술 스택 (요약)

- Unity 6000.5.1f1 / URP **Deferred+** + GPU Resident Drawer + Occlusion Culling
- New Input System · Cinemachine · VFX Graph · UI Toolkit · Awaitable · Mecanim(Animator, Generic 리그)
- 어셈블리 레이어: Core / Data / Systems / Gameplay / UI / Tests
- 패턴: 타입 EventBus · ServiceLocator · 상태기계 · Object Pooling(적·투사체·젬·VFX 전부)
- **성능:** 동시 적 210체 @ 5.2ms(193fps)
- 네비게이션: 커스텀 **Flow Field**(NavMesh/A* 미사용, 선호속도 소스) + **PBD 군중 솔버**(`CrowdSolver`, 적-적/적-벽 통합 제약) + 해석적 벽 충돌(`ObstacleField`)
- 자동화 테스트 70개(EditMode 57 + PlayMode 13)

## 14. 현재 한계 / 미구현 (논의 거리)

1. **아트 일부 적용** — **플레이어 = Crystal 캐릭터 + Idle/Walk 애니메이션(Mecanim) 적용 완료.** 적은 아직 프리미티브 캡슐(수천 마리 크라우드 애니는 Animator 비용상 VAT+GPU 인스턴싱 예정). Leaf 캐릭터는 엘리트/보스용으로 보류.
2. **무기 1종** — MagicBolt만. 장판·근접·다중 투사체 등 미구현
3. **웨이브 단조** — 시간대별 난도 곡선·보스 없음
4. **카드 UI 기능 위주** — 아이콘·등급·보유현황·리롤 없음(회색 버튼)
5. **최대 체력 업그레이드** — `RunStats`만 갱신, 현재 체력 즉시 회복은 미적용
6. **사운드 없음**
7. **메타 progression/세이브 없음** (이번 프로토타입 범위 밖)
